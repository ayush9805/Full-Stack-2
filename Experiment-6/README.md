## ðŸ§ª Experiment 2: Login Validation
A strict client-side validation form that provides real-time feedback to the user before allowing a server submission.

### Key Features
* **Custom Password Toggle:** Includes a professional, custom SVG "eye" icon to toggle password visibility (without default HTML button borders/backgrounds).
* **Dynamic Error Handling:** Displays specific, red error messages immediately below the respective input fields when validation fails. The input boxes also highlight red to indicate errors.
* **Strict Email Validation (Regex):**
  * Must contain an `@` symbol.
  * Must end with a valid domain (e.g., `.com`, `.in`, or a 2-3 letter country code).
  * Prevents edge-case bypassing (e.g., rejecting inputs like `abcd.com@example.com`).
* **Strict Password Validation:**
  * Must start with a Capital Letter.
  * Must contain at least one number.
  * Must contain at least one special character.
  * Must be a minimum of 5 characters long.
